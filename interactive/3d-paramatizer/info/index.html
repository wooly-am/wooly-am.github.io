The goal of this project is to provide an interesting way to control Oscillators and Filters in a unique way: with 3d Models!

My aims were to:

  • Create a DAW-Like interface where users can experiment
  • Visualize “Parameterization” in a satisfying way

How to Use:

To begin, click the “Upload 3d model” and choose a file of your choice. Currently only OBJ files are supported, and simpler, low-poly models are preferred.

Now you should see a small scene with the model you selected against a purple and black background. If you need to reset for any reason, click the x in the top right corner of the page. Use the < > buttons to rotate your model as needed. You can experiment by clicking any of the buttons on the bottom, which creates an audio node. Nodes can be removed by clicking the x on the top right corner of the node.

To modulate a parameter of a node, select an option from the drop down menu on the parameter’s row, next to the slider. More sliders should appear below. Note that to make the modulation audible, “Max” must be greater than 0. out

Design and Methodology

    This webapp can be understood as a series of "wrapper" classes for WebAudio's AudioNode and AudioParam classes. These classes allow for easy implementation of more types of oscillators and filters.

        I’ve used the singleton design pattern to create a playstate that handles scene initialization, audio initialization, and keeps track of node instances. This also allows for a settings panel that can be changed at runtime, by creating a new playstate. Global objects are generally stored in this class, such as the master limiter and the oscillator bus, which provides a single node that every oscillator outputs to.

        Another singleton is SceneHandler, which handles the 3d scene and animation. It contains a trail class that is set to the right position at the right time and fades out, creating a trailing effect.
